    Git命令的使用和项目的上传

    Yan H Qi 5/21/2023 git

    温馨提示

    部分素材来自网络，若不小心影响到您的利益，请联系我们删除。

    使用git命令上传项目的过程

    • 第一步：在GitHub上面创建一个仓库。(如果仓库存在就进入下一步)
    点击右上角加号>>点击 new repository>>输入仓库名>>点击create repository（在这里仓库就创建成功）
    • 第二步：将静态网页代码上传到 GitHub 存储库。（如下代码）
    # 初始化本地仓库（如果需要）
git init

# 添加静态网页代码到本地仓库
git add .

# 提交代码到本地仓库
git commit -m "Initial commit"

# 关联本地仓库与 GitHub 存储库
git remote add origin <GitHub 仓库的 URL>

# 将代码推送到 GitHub
git push -u origin master
如果出现问题
第一步：git pull --rebase origin master
第二步：git push origin main
    • 第三步：配置项目的 GitHub Pages 设置。
    1.进入 GitHub 存储库的页面。
2.点击上方菜单中的「Settings」选项。
3.在页面左侧导航中选择「Pages」。
4.在「Source」部分选择要部署的分支（通常是 master 分支）。
5.选择项目的根目录作为部署路径。
6.点击「Save」保存设置。
    • 第四步：等待 GitHub 完成部署。
    1.GitHub Pages 将自动构建并部署您的静态网页。
2.在部署设置页面的「Environments」部分，您可以查看部署的状态和生成的网页的 URL。

    git常用命令的使用

    1.git init：初始化一个新的 Git 仓库。
例如：git init

2.git clone：克隆（下载）一个远程 Git 仓库到本地。
例如：git clone <repository_url>

3.git add：将文件添加到暂存区（即将文件纳入版本控制）。
例如：git add <file_name>      # 添加指定文件
     git add .               # 添加所有文件

4.git commit：提交暂存区的文件到本地仓库。
例如：git commit -m "Commit message"

5.git push：将本地仓库的提交推送到远程仓库。
例如：git push <remote> <branch>

6.git pull：从远程仓库获取最新的代码并合并到本地仓库。
例如：git pull <remote> <branch>

7.git status：查看仓库的状态，显示已修改、已暂存或未跟踪的文件。
例如：git status

8.git branch：管理分支，列出、创建或删除分支。
例如： git branch                      # 列出所有分支
      git branch <branch_name>        # 创建新分支
      git branch -d <branch_name>     # 删除分支

9.git checkout：切换分支或还原文件。
例如：git checkout <branch_name>      # 切换到指定分支
      git checkout -b <branch_name>   # 创建并切换到新分支
      git checkout -- <file_name>     # 还原文件为最近一次提交的状态

10.git merge：将一个分支的更改合并到当前分支。
例如：git merge <branch_name>

11.git log：查看提交历史记录。
例如：git log

12.git remote：管理远程仓库。
例如：git remote add <remote_name> <remote_url>   # 添加远程仓库
      git remote -v                               # 查看远程仓库列表
