    hook中的useState

    Hook 是一些可以让你在函数组件里”钩入“React state 及生命周期等特性的函数。

    useState来自react，需要从react中导入，他是一个hook；

    ​ 参数：初始化值，如果不设置为undefined；

    ​ 返回值：数组，里面有两个元素；

    ​ 元素一：当前状态的值（第一次调用为初始化值）

    ​ 元素二：改变状态值的函数,触发函数会完成两件事

    ​ 1.调用setCount，设置一个新的值；

    ​ 2.组件重新渲染，根据新的值返回DOM结构

    规则：只能在函数最外层调用hook。不要在循环、条件判断或子函数中调用

    ​ 只能在react函数组件中调用hook，不要在js函数中调用

    jsx=》React.createElement[这个得到的是一个React元素]=》ReactElement{真实DOM}： jsx的执行过程

    hook中的useEffect

    Effect可以完成class中生命周期中功能

    网络请求、手动更新dom，事情监听，都可以使用Effect来完成

    **例：**在class组件中的编写，某些副作用的代码，我们需要在componentWillUnmount中进行清除

    ​ 比如：事件总线（eventbus）或Redux中动手调用subscribe 这些都需要在摧毁生命周期函数中取消订阅

    useEffect传入的回调函数A本身可以有一个返回值，这个返回值是另外的一个回调函数B。

    这个返回值的回调函数是effect可选的清除机制，每一个effect都可以返回一个清除函数

    Effect性能优化

    默认情况下，useEffect的回调函数会在每次渲染时都重新执行，会导致两个问题

    ​ 1.某些代码我们只希望执行一次即可，类似于componentDidMount和componentWillUnmount中完成的事情，（网络请求，订阅和取消订阅）

    ​ 2.多次请求会导致一定的性能问题

    我们如何决定useEffect在什么时候应该执行和什么时候不应该执行：useEffect实际上有两个参数

    ​ 参数一：执行的回调函数；

    ​ 参数二：该useEffect在那些state发生变化时，才重新执行（受谁的影响）

    如果你执行只运行一次的effect，一个函数我们不希望依赖任何内容时，也可以传入一个空的数组 [];

    hook中的useMemo

    把“创建”函数和依赖项数组作为参数传入 useMemo，它仅会在某个依赖项改变时才重新计算 memoized 值。这种优化有助于避免在每次渲染时都进行高开销的计算。

    如果没有提供依赖项数组，useMemo 在每次渲染时都会计算新的值。相当于性能没有改变。

    如果提供了一个空数组，useMemo在每次渲染时不会计算新的值，直接拿原来的值。性能提高了但是自己想要的值没有拿到

    如果提供了依赖项数组，useMemo在每次渲染都会计算新的值，性能也改变了。

    hook中的useCallback

    useCallback 是来记住一个回调，但由于内部函数必须经常重新创建，记忆效果不是很好。

    useCallback 是一个允许你在多次渲染中缓存函数的 React Hook。

    useCallback会返回一个函数的（记忆的）值；

    在依赖不变的情况下，多次被定义的时候，返回的值是相同的

    参数 （一个回调函数和一个数组{数组决定这个函数改不改变}）

    • fn：想要缓存的函数。此函数可以接受任何参数并且返回任何值。React 将会在初次渲染而非调用时返回该函数。当进行下一次渲染时，如果dependencies` 相比于上一次渲染时没有改变，那么 React 将会返回相同的函数。否则，React 将返回在最新一次渲染中传入的函数，并且将其缓存以便之后使用。React 不会调用此函数，而是返回此函数。你可以自己决定何时调用以及是否调用。

    • dependencies：有关是否更新 fn 的所有响应式值的一个列表。响应式值包括 props、state，和所有在你组件内部直接声明的变量和函数。如果你的代码检查工具 配置了 React (opens new window)，那么它将校验每一个正确指定为依赖的响应式值。依赖列表必须具有确切数量的项，并且必须像 [dep1, dep2, dep3] 这样编写。React 使用 Object.is (opens new window) 比较每一个依赖和它的之前的值。

    useCallback性能优化的点：

    1.当需要将一个函数传递给组件时，最好使用useCallback进行优化，将优化之后的函数，传递给子组件

    进一步优化：当count发生变化的时，也是用同一个函数

    做法一：将count依赖去掉，缺点：闭包缺陷；（函数返回的一直是同一个值）

    做法二：useRef，在组件多次被渲染时，返回的是同一个值

    const [count,setCount]=useState(0);

const countRef=useRef();		//这个hook返回的值不会变
countRef.current=count			//这里每次拿到的都是最新的count
const increment=useCallback(function foo(){		//这里每次调用都会创建新的函数：例如第一次foo1。第二次foo2
    setCount(countRef.current+1)
},[])			//没有依赖，所以一次用的是这个函数foo1,
//我们把这个increment这个函数传给一个组件的时候，count的值改变了，这个组件也不会重新渲染

<App increment={increment}></App>

    hook中的useReducer的使用

    useReducer仅仅是useState的一种代替方案：

    1.在某种场景下，如果state的处理逻辑比较复杂，我们可以通过useReducer来对其进行拆分；

    2.或者这次修改的state需要依赖之前的state时，也可以使用；

    使用方法：

    第一步：

    定义一个reducer()函数

    function reducer(state,action){
    switch(action.type){
        case "increment":
            return {...state,counter:state.counter+1}
        case "decremet":
            return {...state,counter:state.counter-1}
        case "add_number":
            return {...state,counter:state.counter-action.num}
        default:
            return state
    }
}

    第二步：

    导入hook中的useReducer

    const [state,dispatch]=useReducer(reducer,{counter:0})			//第一个参数：前面定一个函数    第二个参数：初始化值


//例子	const [state,dispatch]=useReducer(reducer,{counter:0,friend:[]}) 	
//像这种state里面不止包含counter，还包含friend，如果要用useState就要定义两次

    第三步：

    在组件中使用

    onclick={e=>{dispatch({type:"increment"})}}		//这里的dispatch派遣一个action，也就是上面函数中的action，{type:"incremet"}这个就是action

    hook中的useRef的使用

    useRef返回一个对象，返回的ref对象在组件的整个生命周期保持不变的。

    最常用的ref时两种用法：

    用法一：引入D OM元素或组件(但是需要class组件)；

    用法二：保存一个数据，这个对象在整个生命周期中可以保存不变的；(useRef()返回的值是一个对象，是不会变的)

    在vue和react中不要直接操作dom（document.querySelect(".app")） //因为你也不知道这个app就一个是你想要的dom

    使用方法：

    第一步：//导入hook中的useRef()
const titleRef=useRef();

第二步：//在元素上添加  ref={titleRef}
<h2 ref={titleRef}> </h2>

第三步：//使用：dom元素在titleRef.current里面
console.log(titleRef.current)

    useEffect会在渲染的内容更新到DOM上后执行useEffect，不会堵塞DOM的更新；

    useLayoutEffect会在渲染的内容更新到DOM上之前执行，会堵塞DOM的更新；

    const [count,setCount]=useState(100)
useEffect(()=>{
    if(count===0){

    }
